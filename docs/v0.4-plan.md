# Reforge Linux v0.4 Plan

## Goal

The v0.4 goal is to prepare a custom MX Snapshot ISO for Reforge Linux.

This milestone is focused on ISO readiness, branding polish, launcher validation, screenshots, and post-snapshot testing. It is not a production release and does not add new profiles.

## Already Working

- Reforge Setup Center works
- File Server profile using Samba is implemented and tested
- Ad-Blocking DNS Server profile using Pi-hole is implemented and tested
- Media Server profile using Jellyfin is implemented and tested
- System installer installs Reforge Linux to `/opt/reforge-linux`
- `reforge-setup` command exists after system install
- Desktop/application launcher exists after system install
- MX Linux 25.2 XFCE x64 VM testing has been completed for the main profiles

## Before ISO Generation

- Run all shell syntax checks
- Pull the latest main branch
- Run `sudo bash scripts/install-reforge-system.sh`
- Confirm `/opt/reforge-linux` contains scripts, profiles, docs, branding, README, and LICENSE
- Confirm `reforge-setup` opens the setup center
- Confirm the application launcher opens the setup center
- Add or select a Reforge wallpaper
- Confirm no personal/private files will be included in the snapshot
- Clean apt cache and temporary downloads
- Save the generated ISO outside the Git repository

## Screenshots Needed

- Jellyfin dashboard

- MX desktop with Reforge folder or launcher
- Future ISO boot screen

## After ISO Generation

- Boot the generated ISO in a fresh VirtualBox VM
- Confirm the desktop loads
- Confirm Reforge branding appears where configured
- Confirm Reforge Setup Center launcher works
- Confirm `reforge-setup` command works
- Test System Info from the setup center
- Test at least one full profile from the generated ISO
- Confirm no private VM user data was included
- Create SHA256 checksum
- Draft release notes with known limitations
- Later test on Dell Inspiron 1464 hardware

## Notes

- ISO files should not be committed to Git
- ISO builds should be attached to GitHub Releases
- Each release should include checksum, release notes, screenshots, and known limitations
- Pi-hole requires a stable IP or DHCP reservation for real network use
- Reforge Linux remains an experimental learning/portfolio project
